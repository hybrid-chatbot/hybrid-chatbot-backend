# 하이브리드 챗봇 Backend + Frontend 로컬 실행 환경 구축 계획

## 프로젝트 배경

이 프로젝트는 3개 레포로 구성된 하이브리드 쇼핑몰 챗봇 시스템이야.
- `hybrid-chatbot-ai` (FastAPI, Python) — AI 의도 분류 서버 (이미 develop 브랜치에서 작업 완료)
- `hybrid-chatbot-backend` (Spring Boot, Java) — 메인 백엔드 API Gateway
- `hybrid-chatbot-frontend` (React) — 사용자 채팅 UI

지금 해야 할 일: **Backend와 Frontend 프로젝트를 로컬에서 실행 가능한 상태로 만들기**

---

## 작업 순서

### 1단계: 프로젝트 현황 파악

먼저 두 프로젝트의 현재 상태를 파악해줘.

**Backend (hybrid-chatbot-backend)**:
- `develop` 브랜치를 사용 중
- 프로젝트 루트의 파일/폴더 구조를 확인해줘 (tree)
- `build.gradle` 또는 `pom.xml` 확인하여 Java 버전, 의존성 파악
- `application.yml` 또는 `application.properties` 확인하여 포트, DB 설정 파악
- `.env` 또는 `env.example` 파일이 있는지 확인
- Dialogflow 관련 설정 (JSON 인증키 경로 등) 확인
- AI 서버(FastAPI) 호출 URL 설정이 어디에 있는지 확인
- MongoDB, Redis, Kafka 연결 설정 확인

**Frontend (hybrid-chatbot-frontend)**:
- `main` 브랜치를 사용 중
- 프로젝트 루트의 파일/폴더 구조를 확인해줘 (tree)
- `package.json` 확인하여 React 버전, 스크립트, 의존성 파악
- API 백엔드 URL 설정이 어디에 있는지 확인 (환경변수 또는 config 파일)
- `vercel.json` 또는 배포 설정 확인

### 2단계: Backend 로컬 실행 환경 구성

파악한 내용 기반으로 아래를 진행해줘:

1. **Java 버전 확인**: `java --version`으로 현재 설치된 Java 버전 확인. Backend에서 요구하는 버전(보통 17 이상)과 맞는지 확인

2. **의존성 설치 및 빌드 테스트**:
   ```
   ./gradlew build -x test
   ```
   또는 Maven이면:
   ```
   ./mvnw package -DskipTests
   ```
   빌드 에러가 나면 원인 파악하고 해결해줘.

3. **환경변수/설정 파일 정리**:
   - `.env.example` 파일이 없으면 만들어줘
   - 최소한 다음 항목이 필요할 거야:
     - `OPENAI_API_KEY` 또는 AI 서버 URL (예: `AI_SERVER_URL=http://localhost:8000`)
     - MongoDB 연결 정보 (예: `MONGODB_URI=mongodb://localhost:27017/chatbot`)
     - Dialogflow 프로젝트 ID 및 인증 관련
     - Redis 연결 정보 (있다면)
     - 서버 포트 (기본 8080)
   - `application.yml`에서 환경변수를 참조하도록 되어있는지 확인하고, 하드코딩된 값이 있으면 환경변수로 분리해줘

4. **외부 서비스 의존성 확인**:
   - MongoDB: 로컬에 설치되어 있는지 확인. 없으면 Docker로 실행하는 명령어를 알려줘:
     ```
     docker run -d --name mongodb -p 27017:27017 mongo:latest
     ```
   - Redis: 필수인지 선택인지 코드에서 확인. 필수면 Docker 명령어 제공:
     ```
     docker run -d --name redis -p 6379:6379 redis:latest
     ```
   - Kafka: 필수인지 확인. 로컬 실행에 꼭 필요하지 않다면 비활성화 방법을 찾아줘
   - Dialogflow: 인증 JSON 키 파일 경로가 어디로 설정되어 있는지 확인. 실제 키가 없어도 서버가 뜨는지, 아니면 키가 없으면 시작 자체가 안 되는지 확인해줘

5. **AI 서버 연동 설정**:
   - Backend에서 AI 서버(FastAPI)를 호출하는 코드를 찾아줘 (보통 RestTemplate 또는 WebClient 사용)
   - AI 서버 URL이 설정 파일의 어느 변수에 있는지 확인
   - `http://localhost:8000`으로 설정되어 있는지 확인하고, 아니면 수정해줘

6. **서버 실행 테스트**:
   ```
   ./gradlew bootRun
   ```
   - 실행이 되면 http://localhost:8080 (또는 설정된 포트)에서 응답이 오는지 확인
   - 에러가 나면 로그를 보고 원인을 파악해줘 (DB 연결 실패, Dialogflow 인증 실패 등)
   - 필수가 아닌 서비스(Kafka 등) 때문에 실행이 안 되면, 해당 설정을 선택적으로 비활성화하는 방법을 적용해줘

### 3단계: Frontend 로컬 실행 환경 구성

1. **Node.js 버전 확인**: `node --version`으로 확인. React 18은 보통 Node 16+ 필요

2. **의존성 설치**:
   ```
   npm install
   ```
   에러가 나면 `--legacy-peer-deps` 옵션 시도

3. **환경변수 설정**:
   - `.env` 또는 `.env.local` 파일에 백엔드 API URL 설정
   - 보통 `REACT_APP_API_URL=http://localhost:8080` 또는 `VITE_API_URL=http://localhost:8080` 형태
   - 코드에서 API URL을 어떻게 참조하는지 확인하고 맞춰줘

4. **개발 서버 실행**:
   ```
   npm run dev
   ```
   또는
   ```
   npm start
   ```
   - http://localhost:3000 (또는 설정된 포트)에서 UI가 뜨는지 확인

### 4단계: 3개 서버 연동 테스트 가이드 작성

모든 세팅이 끝나면, 다음 내용을 포함한 `LOCAL_SETUP.md` 파일을 Backend 레포 루트에 만들어줘:

```markdown

- Java XX
- Node.js XX
- Python 3.12
- Docker (MongoDB, Redis용)
- OpenAI API Key
- (선택) Google Dialogflow 인증 JSON

## 1. 외부 서비스 실행
(Docker 명령어)

## 2. AI 서버 실행 (Port: 8000)
(hybrid-chatbot-ai 실행 방법)

## 3. Backend 실행 (Port: 8080)
(hybrid-chatbot-backend 실행 방법)

## 4. Frontend 실행 (Port: 3000)
(hybrid-chatbot-frontend 실행 방법)

## 5. 동작 확인
(curl 또는 브라우저에서 테스트하는 방법)

(흔한 에러와 해결법)
```

### 5단계: 문제가 있으면 보고

위 과정에서 발견한 문제점이나 개선할 점이 있으면 정리해줘:
- 빌드가 안 되는 부분
- 하드코딩된 API 키나 URL
- .gitignore에 빠진 항목
- 불필요하게 커밋된 파일 (node_modules, build 폴더 등)
- 코드에서 발견한 잠재적 이슈

## 중요 참고사항

- Backend와 Frontend 레포는 `hybrid-chatbot`이라는 GitHub organization에 있어
- Backend는 `develop` 브랜치, Frontend는 `main` 브랜치에서 작업
- AI 서버는 별도 레포(`jhj1819/hybrid-chatbot-ai`)에 있고, `develop` 브랜치에서 이미 작업 완료된 상태
- Dialogflow 인증키가 없어도 AI 서버(FastAPI)만으로 의도 분류가 가능한 구조야 — Dialogflow 없이도 기본 동작은 되어야 해
- 실행이 안 되는 부분은 무리하게 고치지 말고, 뭐가 문제인지 정리해서 알려줘. 내가 판단하고 진행할게
